# AI Capability Commons Curriculum

Version: 1.0  
Default cohort: 12 weeks, 90 minutes per week, optional 60-minute practice lab  
Audience: adult learners, community college students, nonprofit staff, job seekers, frontline workers, and small-business operators

## Program Promise

Participants learn to use AI responsibly in real work. Each learner graduates with:

- A personal AI use statement
- A prompt and verification library
- A workflow map
- A portfolio project tied to a real community or employer workflow
- A recommended apprenticeship or role pathway

## Core Learning Outcomes

By the end of the cohort, learners can:

- Explain where AI helps, where it fails, and where human judgment remains necessary.
- Turn vague work requests into clear AI-assisted tasks with context, constraints, and review criteria.
- Verify factual claims and document uncertainty.
- Map a workflow and identify safe opportunities for AI assistance.
- Improve one real workflow and present the before/after evidence.
- Discuss AI use with a manager, instructor, client, or community partner using plain language.

## Weekly Plan

### Week 0: Cohort Setup and Baseline

Goal: Make participation accessible and measurable before instruction begins.

Activities:

- Learner intake and access-needs check
- AI tool access check
- Baseline confidence and task fluency survey
- Project partner workflow selection
- Portfolio folder setup

Artifact:

- Baseline survey
- Access plan
- Cohort charter

Facilitator note:

- Week 0 should remove friction. Do not assume learners have accounts, devices, quiet work time, or confidence.

### Week 1: AI, Agency, and the Work Ahead

Goal: Establish a grounded relationship with AI.

Activities:

- Discuss where AI is already showing up in work and learning
- Name common fears and hopes
- Compare "replacement" framing with "capability" framing
- Introduce disclosure, privacy, and human approval norms

Artifact:

- Personal AI use statement

Practice:

- Ask an AI tool to explain a task the learner already knows. Identify what is useful, shallow, wrong, or missing.

### Week 2: Prompting as Work Design

Goal: Teach prompting as structured thinking, not magic wording.

Activities:

- Practice the Context, Task, Constraints, Example, Review pattern
- Rewrite weak prompts into useful work briefs
- Build reusable prompts for learner roles

Artifact:

- Prompt pattern library

Practice:

- Create one prompt for drafting, one for critique, one for summarizing, and one for planning.

### Week 3: Verification and Source Discipline

Goal: Build habits for checking outputs.

Activities:

- Identify hallucination risk
- Trace factual claims to sources
- Mark uncertainty
- Practice source comparison
- Discuss when AI is not appropriate

Artifact:

- Verification checklist

Practice:

- Have AI summarize a public article, then compare the output to the source and correct errors.

### Week 4: Workflow Mapping

Goal: Turn a real task into an improvement opportunity.

Activities:

- Map steps, handoffs, inputs, outputs, decisions, and risks
- Identify candidate AI assists
- Define approval points
- Estimate before-state time and pain points

Artifact:

- Workflow map and risk notes

Practice:

- Map one recurring task from the learner's work, school, volunteer, or job search life.

### Week 5: Role Lab, Communication and Operations

Goal: Use AI for everyday work without losing voice, accuracy, or accountability.

Activities:

- Draft and revise emails
- Summarize meeting notes
- Create checklists and standard operating procedures
- Triage requests using escalation rules

Artifact:

- Operations workflow card

Practice:

- Convert rough notes into an action summary with owners, dates, and open questions.

### Week 6: Role Lab, Data, Research, and Decisions

Goal: Use AI to structure information and support decisions.

Activities:

- Turn messy information into comparison tables
- Generate research questions
- Produce a decision memo with caveats
- Practice reading AI outputs skeptically

Artifact:

- Evidence memo

Practice:

- Compare three options for a real decision and document what still needs human review.

### Week 7: Accessibility and Inclusion by Default

Goal: Make AI-assisted work usable for more people.

Activities:

- Check readability
- Translate jargon into plain language
- Consider language access
- Review disability access
- Identify whose experience is missing from testing

Artifact:

- Inclusive use review

Practice:

- Revise one document for plain language, screen reader structure, and multiple reading levels.

### Week 8: Automation with Guardrails

Goal: Introduce automation only where responsibility is clear.

Activities:

- Distinguish assistive AI from delegated automation
- Map approval and reversal points
- Discuss privacy boundaries
- Create "do not automate" rules

Artifact:

- Guardrail plan

Practice:

- Design a low-risk automation for a recurring task and explain where humans remain in the loop.

### Week 9: Portfolio Sprint One

Goal: Build the first version of a real workflow improvement.

Activities:

- Choose a partner workflow
- Define user, problem, current state, desired state, and constraints
- Build prototype prompts, templates, or process docs

Artifact:

- Project brief and prototype

Practice:

- Run the prototype on a realistic example and document what broke.

### Week 10: Portfolio Sprint Two

Goal: Test, revise, and measure.

Activities:

- Conduct stakeholder feedback session
- Revise based on evidence
- Document before and after
- Identify risks that should block or slow adoption

Artifact:

- Test notes and before/after evidence

Practice:

- Prepare a five-minute demo with one honest failure.

### Week 11: Apprenticeship Handoff

Goal: Translate portfolio work into role language.

Activities:

- Write a role brief
- Practice explaining AI use in an interview or manager conversation
- Identify next learning steps
- Discuss apprenticeship expectations

Artifact:

- Role brief and portfolio narrative

Practice:

- Record or rehearse a two-minute portfolio explanation.

### Week 12: Demo Day and Local Playbook

Goal: Share evidence and make the next cohort easier.

Activities:

- Present portfolio projects
- Collect stakeholder feedback
- Complete post-cohort survey
- Update local playbook
- Invite next cohort partners

Artifact:

- Demo
- Cohort report
- Next cohort plan

## Core Rubric

Score each portfolio project from 1 to 4:

1. Task clarity: The learner explains the human outcome, workflow, and user.
2. AI use quality: The learner gives context, constraints, examples, and review criteria.
3. Verification: The learner checks factual claims and documents uncertainty.
4. Guardrails: The learner identifies privacy, approval, and escalation rules.
5. Usefulness: The project saves time, improves quality, improves access, or makes a process easier to repeat.
6. Communication: The learner can explain what changed in plain language.

## Default Role Pathways

- AI-enabled operations assistant
- Student services navigator
- Marketing and content operations apprentice
- Data and story analyst
- Nonprofit capacity builder
- Small-business process improver
